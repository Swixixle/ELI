## A) Fix the port collision in the dev workspace (so you can keep iterating)

### 1) Stop the process that’s already holding 5000

In Replit, the most common cause is an already-running “workflow” (preview/server) that never stopped.

**Fastest:**

* Click **Stop** in the Replit run panel (the running process).

If you want the shell method (works even when UI is weird):

```bash
kill -9 $(lsof -t -i :5000) 2>/dev/null || true
npm run dev
```

If `lsof` isn’t available:

```bash
ss -lptn | grep :5000
# then kill -9 <PID>
```

### 2) Prevent future collisions (recommended)

Make sure `server/index.ts` binds to env PORT:

```ts
const port = Number(process.env.PORT) || 5000;
app.listen(port, "0.0.0.0", () => console.log(`listening on ${port}`));
```

This makes Replit deployments happier and keeps dev flexible.
