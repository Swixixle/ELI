AI↔AI Protocol v0.1 — Envelope-First Exchange (EFX)

Design goals
	1.	No free-text authority transfer (text can exist, but cannot carry “meaning authority” alone).
	2.	Measurements cannot exist without envelopes (your M5 rule).
	3.	Refusal is a valid terminal message type (not an error).
	4.	Downstream AIs must explicitly acknowledge constraints before using outputs.
	5.	Auditable: every step yields a traceable object.

⸻

1) Message Types

A. REQUEST

“Here is what I want you to do, and here is what I am allowed to use it for.”

Required fields:
	•	request_id
	•	sender_agent_id, receiver_agent_id
	•	speech_act (enumerated)
	•	allowed_uses[] (enumerated)
	•	context_bundle (references + hashes, not raw blobs if possible)
	•	decision_time_anchor (optional; required in SRE flows)
	•	constraints (optional; required if the target system requires S2)

B. RESPONSE_PERMITTED

“Here is a measurement/output with its envelope, and what it can be used for.”

Required:
	•	request_id
	•	response_id
	•	payload (may contain text, but not authoritative without envelope)
	•	measurement (value + envelope)
	•	proof (gate results / locked strata references / hashes)
	•	warnings[] (optional)

C. RESPONSE_REFUSAL

“I refuse, and here is the formal reason.”

Required:
	•	request_id
	•	response_id
	•	refusal:
	•	code
	•	governing_axiom
	•	failed_gate (e.g., S1 lock, S2 lock, entitlement triad, envelope)
	•	human_action_required (boolean)
	•	required_inputs[] (what would make it admissible)
	•	proof (what was checked, what was missing)

D. ACKNOWLEDGMENT

“I acknowledge your envelope and I bind myself to its authorized uses.”

Required:
	•	response_id
	•	ack_id
	•	acknowledged_envelope_hash
	•	intended_use (must be in authorized_uses[])
	•	no_prohibited_use_attestation: true

Without an ACK, downstream use is prohibited in the protocol.

3) Core Rules (Protocol “Constitution”)

Rule 1 — Envelope primacy

If measurement.value exists, measurement.envelope MUST exist.

Rule 2 — Fail-closed

If any gate fails, only RESPONSE_REFUSAL is allowed.

Rule 3 — No implicit reuse

A downstream agent may not use or transform a measurement until it sends an ACKNOWLEDGMENT.

Rule 4 — Scope binding

Downstream outputs MUST carry forward:
	•	source_measurement_id
	•	source_envelope_hash
	•	intended_use
	•	and must refuse if the intended use is prohibited.

Rule 5 — No laundering via paraphrase

If a system refuses or prohibits an action, the downstream agent cannot restate the forbidden thing in different words. It must either:
	•	refuse, or
	•	request admissible inputs.

5) Handshake sequence (what actually happens)
	1.	A → B: REQUEST
	2.	B → A: RESPONSE_PERMITTED or RESPONSE_REFUSAL
	3.	If permitted, A → B: ACKNOWLEDGMENT
	4.	Only after ACK may B → A provide any derived artifacts (plots, comparisons, summaries) tied to intended_use.

This prevents “silent reuse” and gives you a clean audit chain.
