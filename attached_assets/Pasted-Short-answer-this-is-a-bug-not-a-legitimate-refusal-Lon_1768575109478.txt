Why this is NOT a refusal

	1.	It is explicit
	2.	It names the reason:
	•	e.g. temporal_boundary
	•	insufficient_prerequisites
	3.	It explains why the question cannot be answered
	4.	It never says “I’m not sure what you mean”

What you got instead is a generic assistant fallback, which violates your Canon.

So this is not epistemic restraint.
It’s state leakage.

Evaluator → readiness check → structured answer

Generic LLM → intent confusion → fallback helper text
