Minimal ACK design (v0.1)

What it does
	•	Prevents silent reuse/laundering by forcing the consuming agent to state intended use
	•	Creates an audit artifact that downstream transforms are allowed only after ACK

Add one endpoint (server)

POST /api/acks

Request body
	•	response_id
	•	acknowledged_envelope_hash
	•	intended_use (must be in envelope.authorized_uses)
	•	no_prohibited_use_attestation: true

Server behavior
	•	verify envelope_hash matches
	•	verify intended_use ∈ authorized_uses
	•	store ACK linked to caseId + measurementId
	•	return ACK_ACCEPTED with ack_id

Add one client rule (hard)

No downstream agent action that transforms/visualizes/compares measurements is allowed unless:
	•	ACK exists for that measurement_id + intended_use
