Right now your useQuery throws on any !res.ok, which forces your “Constitutional Refusal” error card, but it’s generic and may be masking structured refusal data.

2A) Edit the queryFn in /cases/:caseId/terrain route component

Replace this:

  throw new Error(responseData.message || responseData.error || "Evaluation failed");

With this exact logic:

const responseData = await res.json().catch(() => ({} as any));

/**
 * Treat constitutional refusals as DATA, not exceptions.
 * Only throw for true server/client errors.
 */
if (!res.ok) {
  // If the API returns a structured constitutional refusal, render it.
  if (res.status === 403 && (responseData.error || responseData.code || responseData.governingAxiom)) {
    return responseData;
  }

  // Otherwise, this is a real error (500, network, malformed JSON, etc.)
  throw new Error(responseData.message || responseData.error || `Request failed (${res.status})`);
}

return responseData;

This one change ensures:
	•	403 refusals get returned as data
	•	only genuine errors throw
