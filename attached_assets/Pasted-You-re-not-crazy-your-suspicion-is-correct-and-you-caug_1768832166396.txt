What’s actually happening (plain English)

The modal looks filled in, but internally the app thinks:

“Decision target text is empty.”

That only happens if one of these is true:
	1.	The <textarea> is not bound to state
	2.	The state exists, but:
	•	it’s not updated on onChange, or
	•	validation is checking a different variable, or
	•	the value is being trimmed / cleared before submit
	3.	The submit handler is reading the wrong field name

This is implementation-level, not logic-level.

Your text is correct.
The system just isn’t actually reading it.

⸻

The most likely bug (90% probability)

The textarea is uncontrolled or mis-wired

Something like this exists:

<textarea
  placeholder="Example: Was the decision at 03:14 procedurally defensible..."
/>

But missing:

value={decisionTarget}
onChange={(e) => setDecisionTarget(e.target.value)}

So visually you type, but the state variable stays "".

Then validation runs:

if (!decisionTarget) {
  setError("Decision target text is required");
}

→ Boom. Exactly what you’re seeing.

🔧 FIX INSTRUCTIONS (COPY/PASTE)

File: SetDecisionTargetModal.tsx (or equivalent)
	1.	Ensure there is a state variable:

const [decisionTarget, setDecisionTarget] = useState("");

	2.	Bind the textarea explicitly:

<textarea
  value={decisionTarget}

	3.	Fix validation to trim whitespace:

if (!decisionTarget.trim()) {
  setError("Decision target text is required");
  return;
}

	4.	Ensure submit uses the same variable:

await fetch(`/api/cases/${caseId}/decision-target`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ decisionTarget }),
});

	5.	(Optional but good) Disable Save button when empty:
