	•	A “truth refusal” system when epistemic entitlement is absent (Parrot Box constraint).  ￼

1) Non-negotiable behaviors (“Honest machine” constraints)

Implement these as hard system invariants, not “best effort”:
	1.	Citations required
	•	Every material factual claim must cite either:
	•	Private canon (doc + version + section + chunk id), OR

	•	If no citation exists, output: “Not in canon / Not supported by sources.” (and stop).
	2.	Outcome-blindness
	•	The assistant must not allow downstream/outcome knowledge to justify what “should have been known” at decision-time; it must enforce temporal admissibility boundaries consistent with ELI’s core rationale.  ￼
	3.	Silence is a valid final output
	•	The system must be allowed to return “No evaluative statement permitted” / “Context insufficient for inference” as final responses (Parrot Box).  ￼
	4.	No invention of numbers, pilots, efficacy, or claims
	•	If the user asks for metrics not grounded in canon or a public dataset, the assistant must refuse and request source input.

2) IP protection requirements (do not leak the product)

You must architect the system so it can be demonstrably correct without revealing proprietary ELI mechanics.

Prohibited disclosures (hard block):
	•	Any internal scoring weights, thresholds, bands, matrices, or “decision engine” mechanics that are not explicitly approved for disclosure.
	•	Any “how it works internally” beyond high-level principles already in canon.

Allowed disclosure:
	•	The epistemic principles: admissibility, outcome-blindness, refusal/silence, governance posture shift.  ￼  ￼
	•	Computation arithmetic, as long as the proprietary parameterization is abstracted (see §4).

3.2 Retrieval policy

	3.	Tier-2 “Draft/Exploratory” (only if user explicitly requests “draft mode”)

4) “Prove the numbers” without exposing IP (audit layer)

You must implement a computation audit subsystem that produces reproducible math while keeping proprietary logic hidden.

4.1 Response contract: mandatory structured output
All answers must return:
	•	answer_markdown
	•	citations[] (private canon chunk refs and/or public sources)

	•	ip_safety_flags[] (e.g., “internal parameter withheld”)

4.2 Calculation proof rules
When the assistant outputs any number derived from computation:
	•	Show inputs with sources (or clearly labeled assumptions).
	•	Show arithmetic steps explicitly (line-by-line).
	•	Show final result and a sensitivity note.

4.3 IP-safe abstraction

	•	Do not reveal the rule.
	•	Instead, present it as a sealed parameter, e.g.:
	•	“Applied an internal governance adjustment factor (withheld as IP) within permitted bounds.”
	•	You may show:
	•	The post-adjustment effect on outputs
	•	The public-data inputs and arithmetic
	•	But not the internal adjustment derivation.
