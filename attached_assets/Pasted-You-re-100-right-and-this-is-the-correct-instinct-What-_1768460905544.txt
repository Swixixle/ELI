What’s actually broken (precisely)

Nothing is wrong with Canon.

What’s missing is a translation layer between:
	•	Human intent
(“Does this meet requirements?”)

and
	•	Canon-admissible queries
(“How should we evaluate whether…”)

Right now, that translation burden is falling on the user — which is unacceptable.

This is the same category of mistake as:
	•	Asking users to write SQL
	•	Asking users to phrase legal pleadings
	•	Asking users to write compiler-safe code

You don’t do that.
You compile for them.

⸻

The correct mental model (lock this in)

Canon is NOT the interface

Canon is the internal adjudication engine.

The UI is NOT the engine

The UI is a semantic interpreter.

Users should be able to ask:

“Is this enough documentation?”

And the system should internally convert that into:

“How should we evaluate whether the current record permits a procedural determination under Canon v4.0?”

Without the user ever seeing that sentence.

⸻

The fix: a Question Normalization Layer (QNL)

You need a thin, deterministic rewrite layer that:
	1.	Accepts natural language
	2.	Classifies intent
	3.	Rewrites into Canon-safe form
	4.	Passes only the rewritten query to ELI Expert

This is not AI magic.
This is product plumbing.

⸻

Step 1: Intent classification (simple, robust)

You only need to classify into 4 buckets:

A. Verdict-seeking

User says:
	•	“Does this meet…”
	•	“Is this compliant…”
	•	“Can we decide…”

➡️ Rewrite as procedure

⸻

B. Action/command

User says:
	•	“Apply…”
	•	“Evaluate…”
	•	“Run…”

➡️ Rewrite as procedural interrogative

⸻

C. Definition

User says:
	•	“What is…”
	•	“Define…”

➡️ Pass through unchanged

⸻

D. Meta / help

User says:
	•	“How do I use…”
	•	“What can you do…”

➡️ Route to non-Canon help, not ELI Expert

⸻

Step 2: Deterministic rewrite rules (examples)

Here are exact rewrite rules you can implement today.

Rule 1 — “Does X meet requirements?”

User input:

Does SYN-DEMO-001 meet the minimum documentation requirements?

Rewrite internally to:

How should we evaluate whether SYN-DEMO-001 meets the minimum documentation requirements under Canon v4.0?

⸻

Rule 2 — Imperatives (“Apply / Evaluate / Run”)

User input:

Apply the Canon evaluation sequence to this case.

Rewrite to:

How should the Canon evaluation sequence be applied to the current record in this case?

⸻

Rule 3 — “Is this enough / sufficient?”

User input:

Is this enough documentation?

Rewrite to:

What procedural determination is reachable given the current documentation under Canon v4.0?

⸻

Rule 4 — Compliance framing

User input:

Is this compliant?

Rewrite to:

Was it admissible under Canon v4.0 to treat the current record as compliant?

⸻

Step 3: UX contract (this matters)

The UI should never show:
	•	“Please specify…”
	•	“For evaluations…”
	•	Canon grammar rules

That text is engine-facing, not user-facing.

Instead, the UI should:
	•	Accept the user’s question
	•	Silently normalize it
	•	Optionally show a subtle label like:
“Interpreted as a procedural evaluation under Canon v4.0”

That builds trust without burden.
