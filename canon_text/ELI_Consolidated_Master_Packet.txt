1. What ELI Is (and Is Not)
ELI (Epistemic Load Index) is a retrospective governance instrument designed to stabilize
adverse-event review by classifying the epistemic conditions under which clinical decisions were made.
It classifies environments, not individuals.

•   Retrospective only: applies after a case independently triggers formal review.

•   Outcome-blind: scoring uses only information meaningfully available at the time; outcome fields are
    prohibited.

•   Environment classification only: not fault, risk, probability, or performance.

•   Accountability separation: deviation analysis is independent; ELI does not mitigate negligence.

•   Anti-surveillance: individual ranking/trending is prohibited; aggregation minimums prevent
    individual inference.

3. Foolproofing and Safety Architecture
Safeguards are enforced in code, not policy.

•   Hindsight Kill-Switch: accountability locked until ELI assessment finalized; post-lock changes
    invalidate the case and create an immutable record.

•   Discovery Firewall: stores no PHI/patient identifiers/clinician names; only anonymized case
    references.

•   Anti-Surveillance Hard-Coding: aggregation minimums make individual-level analysis
    mathematically impossible.

•   Standardized Language Lock: free-text narrative prohibited in exported artifacts; pre-vetted
    phrases only.

•   No-Outcome Filter: no outcomes/resolution fields; preserves epistemic boundary.

•   Misuse Auto-Suspension: detects prohibited patterns (ranking, rescoring) and suspends reporting.

•   Procedural Integrity Score: flags governance-compromised cases and restricts external use.
4. Clinician Context Capture (Optional, Privileged)
Optional clinician debrief captures constraints/trade-offs at the time of decision to counter hindsight
bias. It must exclude identifiers and remain within protected peer review / QI where applicable.
5. The Five-Lens Evaluation Framework
   Lens                          Definition / what a “5” looks like
   Lens 1 — Buyer Reality        Named economic owner, budgeted pain, and a defensible reason to buy now.
   Lens 2 — Legal & Risk Defensibility
                              Reduces exposure; avoids new discoverable artifacts; defensible scope boundaries.
   Lens 3 — Operational Feasibility
                               Integrates into existing workflow; low drag; retrospective-only; minimal IT risk.
   Lens 4 — Political ResistanceNon-punitive commitments; clean exits; misuse protection; minimal power disruption.
   Lens 5 — Asymmetric Value Avoided downside/tail-risk framing; cost small vs prevented escalation delta.
6. CEO-Level Readiness Checklist (Summary)
CEO readiness is demonstrated when ELI is framed as a governance instrument that can terminate
cleanly, operates within existing protections, and plausibly pays for itself via avoided cost amplification.
7. Columbus Pilot: Governance Rider Framing
•   Hindsight control: finalize ELI scoring before outcomes are reviewed.

•   Artifact suppression: reduce informal narrative/documentation sprawl.

•   Posture stabilization: prevent escalation into higher-cost litigation posture.

•   Clean exit: pre-committed termination/off-boarding if break-even conditions aren’t observed.
8. Columbus Pilot Metrics (Retrospective-Only)
   Metric                            Definition
   Outcome-Blind Compliance Rate     Percent of cases finalized before outcome review (target ≥90%).
   Artifact Count per Case           Export artifact count; goal is compression/discipline.
   Review Cycle Time                 Time to sign-off; goal is compression without added drag.
   Escalation Count Not Increased    Stabilization of escalation frequency vs baseline.
   Misuse Attempt Count              Any attempts at ranking/rescoring; should be 0; triggers response.
   Procedural Integrity Violations   Count of governance-compromised cases (outcome leakage, post-lock edits).

10. CFO Decision Engine (Profit Calculator)
   Input                                            Symbol   Meaning
   Annual file volume                               N        Annual malpractice/liability files managed.
   Avg defense spend per file                       D        Average legal + administrative defense spend per file.
   Escalation rate                                  E        Fraction escalating to high-cost posture.
   Escalation delta                                 ∆E       Incremental cost per escalated file.
   System annual cost                               C        Annual license + ops cost for the rider.


   Item                         Formula
   Baseline Exposure            Baseline = (N × D) + (N × E × ∆E)
   Efficiency Factor            F = MIN(10%, 5%×OB + 2.5%×AV + 2.5%×RC)
   Observed Benefit             Benefit = (Baseline × F) + (EscalationsAvoided × ∆E)
   Net Position                 Net = Benefit − C
   Decision Status              PROCEED if Net ≥ 0; OBSERVE if −0.25×C < Net < 0; TERMINATE if Net ≤ −0.25×C
11. Procedure to Run the Model
1 Collect 3–5 years internal risk data for N, D, E, ∆E.
2 Enter Layer A inputs (no benchmarks).
3 Run pilot (5–10 retrospective cases).
4 Verify governance conditions; flip toggles only after verification.
5 Review Net Position and Decision Status; execute termination protocol if required.
